# 竞争调研:三层降级 + 人工验证清单

目的:判断"这个机会有没有人已经在做、做到什么程度、还有没有差异化空间"。三层按可用性降级,**无论用哪层,人工验证清单都必须附在报告里**——平台站内数据(销量、真实定价、评价)只有人工登录才能看准。

## 层 1:浏览器直查(有浏览器工具且用户授权时)

适用:本地 Claude Code + 浏览器扩展/Playwright,或用户明确授权用其登录态访问。

对每个机会查两个平台:

- **闲鱼(goofish.com)**:搜"<项目名>"、"<项目名> 部署"、"<需求词> 服务"。记录:在售数量、价格分布、销量可见的头部卖家、商品形态(卖文件还是卖服务)。
- **淘宝**:同样关键词。淘宝有店铺的服务商定价通常是闲鱼个人卖家的 2-5 倍,用来标定价格上限。

注意:只做只读浏览,不代替用户下单、留言、上架。访问失败(风控/验证码)不要重试硬闯,降级到层 2。

## 层 2:公开网络搜索(默认可用)

用 WebSearch 测**内容饱和度**作为竞争代理指标:

| 搜索 | 解读 |
|---|---|
| `<项目名> 教程` / `<项目名> 部署` | 中文教程 >5 篇且是近一年的 → 信息差已薄,纯教程机会关闭;但代装服务仍可行(看得懂教程≠愿意动手) |
| `<项目名> 闲鱼` / `<项目名> 淘宝` | 有人讨论买卖 → 说明已有市场,去层 3 验证价格 |
| `<项目名> site:xiaohongshu.com` / `zhihu.com` | 小红书/知乎有种草内容 → 需求端有热度 |
| `<需求词> 服务` | 直接搜服务供给 |

饱和度结论只分三档:**空白**(几乎搜不到中文内容)/ **有内容无服务**(教程多但没人卖交付)/ **已有服务**(明确有人在卖)。"有内容无服务"是最好的进入点:需求被教育过,交付没人做。

## 层 3:人工验证清单(始终生成)

为每个机会生成一张 10 分钟人工核查卡,格式:

```markdown
### 人工验证:<机会名>
在闲鱼 App 依次搜索并记录(预计 10 分钟):
1. 搜「<关键词1>」「<关键词2>」「<关键词3>」
2. 记录:
   - 在售商品数:____(>30 个视为红海)
   - 价格带:最低 ____ / 众数 ____ / 最高 ____
   - 销量最高的 3 个商品:卖的是文件还是服务?有没有"包售后/包更新"?
   - 差评关键词:____(差评 = 现有卖家没做好的交付,是你的切入点)
3. 判定:
   - [ ] 空白 → 小步试:先上 1 个 SKU 验证询单量
   - [ ] 有人卖文件没人卖服务 → 用服务差异化进入
   - [ ] 已有服务卖家且评价好 → 除非有明确差异化,放弃
```

## 竞争结论如何写进报告

每个机会的竞争字段只写四项:调研层级(1/2/3)、饱和度档位、证据(链接或搜索词)、进入建议(空白试错 / 服务差异化 / 放弃)。不要写"竞争较大"这类无法行动的模糊结论。
